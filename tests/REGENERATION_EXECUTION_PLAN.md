# Fresh Generated Test Execution Plan

Date: 2026-04-30

Goal: remove the checked-in generated source tree, regenerate tests from the macro-first generator, run them, and improve the generator toward meaningful autogenerated coverage.

Steps:

1. Inspect the current generator and source APIs that will become the only source of regenerated tests.
2. Strengthen generator defaults before deletion so fresh output has meaningful values and multiple scenarios where practical.
3. Delete `tests/generated/src` as requested.
4. Run `haxe compile.hxml` from `tests` to regenerate runner, support modules, catalog, and test suites.
5. Run focused generated tests, then broader test runs.
6. Fix generator issues at the planning/provider/emitter level instead of hand-editing generated tests.
7. Record what remains weak or intentionally deferred in the final report.

Rules for this pass:

- Do not hand-edit regenerated test suites except through generator changes.
- Preserve source code behavior unless tests reveal a root cause that blocks generated tests.
- Prefer meaningful generated scenarios over merely compiling smoke calls.
- Keep failing tests visible when they indicate real generator limitations.
